# Registro Febo AI - 2026-05-18

## Resumen

Se construyo y dejo desplegado el MVP inicial de Febo AI como reemplazo minimo de Hariaz/Aria para atencion comercial por WhatsApp.

El MVP quedo funcionando con:

- App Next.js desplegada en Vercel.
- Base Neon Postgres cargada con contactos importados desde Hariaz.
- Inbox interno multiusuario basico.
- Login interno propio.
- PWA instalable desde el celular.
- Agente Febo AI conectado al prompt operativo v1.
- Repo GitHub creado y conectado a Vercel.

## URLs

- Produccion Vercel: https://febo-ai.vercel.app
- GitHub: https://github.com/Febecos/febo-ai
- Proyecto Vercel: https://vercel.com/pozeroagro/febo-ai

## Stack decidido

Se decidio centralizar Febo AI en:

- **Vercel** para app, API routes, webhook y PWA.
- **Neon Postgres** para contactos, conversaciones, mensajes, usuarios, asignaciones y eventos.
- **OpenAI API** para el agente conversacional.
- **Meta WhatsApp Cloud API** para conectar WhatsApp en una etapa posterior.

Supabase dejo de ser dependencia del MVP de Febo AI.

## App / Inbox

Se construyo un inbox basico con:

- Login interno por email + codigo interno.
- Metricas superiores.
- Lista de conversaciones.
- Detalle de conversacion.
- Acciones:
  - tomar conversacion
  - asignar vendedor
  - cambiar estado
  - pausar/activar IA
- Consola lateral para probar IA.
- Manifest e icono para instalar como PWA.

Estados disponibles:

- `open`
- `waiting`
- `quoted`
- `hot`
- `handoff`
- `closed`
- `lost`

Se corrigio la ergonomia inicial:

- Los botones de accion pasaron arriba dentro del detalle.
- La lista tiene scroll propio.
- En pantallas chicas el detalle queda antes que la lista.
